# Why VULK

This page explains where VULK fits in the AI builder landscape.

## The real market split

Most tools optimize for one of these:
- Fast UI generation
- IDE coding assistance
- Prompt-to-prototype demos

VULK is optimized for **prompt to production workflow**:
- build
- iterate
- deploy
- operate

## VULK vs typical AI builders

| Capability | Typical AI Builder | VULK Direction |
|---|---|---|
| UI generation | Strong | Strong |
| Full product flow | Partial | Core focus |
| Backend-aware generation | Limited | First-class path |
| Deploy workflow | Basic | Built into core workflow |
| Product ops (billing/admin/usage) | Usually external | Integrated platform direction |
| Multi-surface output (web/mobile/game) | Usually narrow | Broad surface strategy |

## Who VULK is for

- Founders building MVP to production
- Product teams shipping new experiments fast
- Agencies delivering multiple client apps
- Builders who need speed without losing architecture control

## Why this matters

Getting a UI draft is no longer the hard part.
The hard part is shipping a real product with deploy, data, and operational control.

That is where VULK is focused.
